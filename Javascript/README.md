# Javascript — Study folder

Structure created to separate solved outputs from theory notes for easy revision.

Folder layout:

- `basics/`
  - `output.md` — your solved code examples (moved from `basics.txt`)
  - `theory.md` — add explanations, complexity, and notes here

- `arrays/`
  - `output.md` — solved array problems (moved from `arrays.txt`)
  - `theory.md` — add explanations and notes here

How to use
----------
1. Open the topic folder (e.g., `basics/`) when revising a topic.
2. Read `output.md` for solved code and `theory.md` for conceptual notes.
3. Add more topic folders (e.g., `strings/`, `datatypes/`) following the same pattern.

Why this helps
---------------
- Keeps code examples and conceptual notes separate for faster revision.
- Easy to locate examples and add more theory without clutter.
